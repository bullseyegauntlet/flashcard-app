## Architecture Summary

**Recommendation:** Vanilla JavaScript (ES6+) with vanilla CSS3.

**Tech Stack:**
- **Frontend:** Vanilla JS + semantic HTML5 + CSS Grid + CSS 3D Transforms
- **State:** In-memory AppState + localStorage (single source of truth)
- **Build:** None (static site served by Netlify)
- **Deployment:** Netlify with GitHub auto-deploy

**Core Modules (6):**
1. **StorageManager** — localStorage adapter (load, save, quota checks)
2. **DeckManager** — CRUD for decks and cards
3. **ProgressTracker** — Calculate progress %, mark cards as known/review, reset
4. **ShuffleEngine** — Non-destructive Fisher-Yates randomization
5. **StudySession** — Manage study flow (current card, flip state, mark, navigate)
6. **UIController** — Screen routing and event handling

**Data Model:**
- **Deck:** `{ id, name, createdAt, cards[] }`
- **Card:** `{ id, front, back, known }`
- **Storage:** localStorage["decks"] = JSON array of Decks

**Key Design Decisions (ADRs):**
1. Vanilla JS over React (simplicity, no build overhead, faster animations)
2. localStorage as single source of truth (offline-first, privacy, no server)
3. Session-only shuffle (fresh randomization each study session, deck order unchanged)
4. Progress stored in card.known flag (atomic updates, simple reset)
5. In-memory AppState for instant reads/renders (fast UX, localStorage writes are fire-and-forget)

**Performance Strategy:**
- Pagination (50 cards/page) for large decks (1000+)
- CSS 3D transforms for smooth card flip (<500ms, GPU-accelerated)
- Lazy-loading on demand (optional compression for v2)
- localStorage quota monitoring (5-10 MB limit ≈ 500-1000 decks)

**Implementation Readiness:**
- Phase 1: Project setup (GitHub, Netlify, directory structure)
- Phase 2: Core modules (StorageManager, DeckManager, ProgressTracker, ShuffleEngine)
- Phase 3: StudySession + UIController
- Phase 4: HTML/CSS (Home, Deck view, Study mode screens)
- Phase 5: Integration & testing (E2E flows, edge cases, performance benchmarks)
- Phase 6: Deploy + docs (Netlify, README, code comments)
