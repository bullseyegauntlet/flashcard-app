## Implementation Details

**StudySession Module:**
- Manages study state: currentCardIndex, isFlipped, isAnswered
- start(deckId): Validates deck, copies cards, initializes
- flip(): Toggles isFlipped state
- markKnown/markReview(): Updates card in DeckManager, saves to localStorage
- advance(): Increments index, resets flip/answered state
- getProgress(): Returns {current, total, percent}
- isComplete(): Checks if past last card

**UI Implementation:**
- Three screens: study, study-complete, plus existing home/deck
- Study screen: Header with deck name + progress, 3D card, action buttons
- Buttons appear/hide based on isFlipped state
- Study complete: Summary with stats + "Study Again" / "Back to Home" options
- Toast notifications for user feedback (success/info types)

**CSS 3D Flip Animation:**
- perspective: 1000px on container
- transform-style: preserve-3d on inner element
- rotateY(180deg) on flip
- backface-visibility: hidden on front/back
- Transition: 0.6s for smooth animation
- Hardware-accelerated (GPU friendly)

**Progress Tracking:**
- Real-time known count updated in header
- Progress % calculated and displayed
- Both updated as user marks cards
- Summary at completion shows final stats
